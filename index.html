HTML

  • head element contains info about webpage
  • body element represents visible content shown to user
  • style element defines the css styles associated with website
  • base element define base URL for a webpage
  • link element connects an external rsource to the HTML document
  • meta element defines metadata such as the character set, description, keywords, author, and viewport
  • script element embeds client-side scripts in the HTML document
  • div elemnt creates a division in the page
  • span element creates an inline division
  • h1 - h6 elements represent level of heading a given text block represents
  • p element represents a paragraph or block of text
  • ul, ol, and li elements represent unordered lists, ordered list, and list items
  • img element contains information about images that are displayed on a webpage
  • br element creates a line of empty space
  • a element, or anchor, creates links to the same webpage or other webpages
  • src attribute defines location, or source, of an image flie
  • alt attribute conatins a text string that describes the appearance and functionality of the image
  • alt text is important for screen readers and other assitive technologies
  • footer elemtn appears ath bottom of page and usually contains author, copoyright, contact, sitemap, and navigation
  • section elements are semantic elements that define a section ina document that contains thematic content

CSS

  • a mragin indicates how much space we want around the outside of an element
  • a padding indicates how much space we want around the content inside an element
  • with CSS you can adjust images in HTML. images are adjusted through display, height,width, margin-left, and margin-right
  • lists and paragraphs can also be formatted with CSS

GIT

    stage and snapshot

  • git status: checks what branch we are currently on
  • git add [file]: add a file as it looks now to your next commit
  • git reset [file]: unstage a file wile retaining changes in working directory
  • git diff: diff of what is changed but not staged
  • git diff - - staged: diff of what is staged but not committed
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot

    • Setup

  • git config - -global user.name "[firstname lastname]": set a anme thats indetifiable for credit when review version history
  • git config - -globale user.email "[valid-email]": set an email address that will be associated with each history maker
  • git config - -global color.ui auto: set automatic command line coloring for Git for easy reviweing

    • setup & init

  • git init: intialize an exisiting directory as a git repository
  • git clone [url]: retrieve an entire repo from hosted location via URL

    • Branch & Merge

  • git branch: list your branches.a* will appear next to the currently active branch
  • git branch [branch-name]: create a new branch at the current commit
  • git checkout: switch to another branch and check it out into your working directory
  • git merge [branch]: merge specified branch's history into current one
  • git log: show all commits in the current branch's history

    • inspect & ompare

  • git log: show commit history for currently active branch
  • git log branchB. .branchA: show the commits on branchA that are not on branchB
  • git log - -folloq [file]: show commits that change file, even across renames
  • git diff branchB. . .branchA: show the diff of what is in branchA that is not in branchB
  • git show [SHA]: show any obkject in git in human-readablke format

    • tracking path changes

  • git rm [file]: delete the file from project and stage the removal for commit
  • git mv [existing-path] [new-path]: change an existing file path and stage the move
  • git - -stat -M: show all commit logs with indication of any paths that moved

    • share & update

  • git remote add [alias] [url]: add a git URL as an alias
  • git fetch [alias]: fetch down all the branches from Git remote
  • git merge [alias]/[branch]: merge a remote branch into your current branch to bring it up to date
  • git push [alias] [branch]: transmit local branch commits to the remot repository branch
  • git pull: fetch and merge any commits from the tracking remote branch

    • Rewrite history

  • git rebase [branch]: apply any commits of current branch ahead of specified one
  • git reset - -hard [commit]: clear staging area, rewrite working tree from specified commit

    • Temporary commit

  • git stash: save modified and stage changes
  • git stash list: list stack-order of stashed file changes
  • git stash pop: write working from top of stast stack
  • git stash drop: discard the changes from top of stash stack
